# ATmosphere Plugin

WordPress plugin that publishes posts to AT Protocol in both `app.bsky.feed.post` (Bluesky) and `site.standard.document` / `site.standard.publication` (standard.site) formats via native OAuth.

## Quick Reference

### Commands

```bash
composer install              # Install PHP dependencies.
npm install                   # Install Node dependencies (wp-env).
composer lint                 # Check PHP coding standards.
composer lint:fix             # Auto-fix PHP issues.
composer test                 # Full PHPUnit test suite (installs test DB).
npm run env-start             # Start WordPress environment.
npm run env-stop              # Stop environment.
npm run env-test              # Run PHP tests via wp-env.
npm run env-test -- --filter=pattern  # Run matching tests.
```

### Running Single Tests

```bash
# Via wp-env:
npm run env-test -- --filter test_function_name

# Via local PHPUnit:
vendor/bin/phpunit --filter test_function_name
```

## Architecture

- **OAuth layer** (`includes/oauth/`): Full PKCE + DPoP + PAR native OAuth flow.
- **API client** (`includes/class-api.php`): DPoP-authenticated PDS requests with nonce retry.
- **Transformers** (`includes/transformer/`): WP → AT Protocol record conversion.
- **Publisher** (`includes/class-publisher.php`): Atomic batch applyWrites for both record types.
- **Admin** (`includes/wp-admin/`): Settings page, meta box, client-metadata REST endpoint.

## Autoloading

Uses Composer's `classmap` autoloader (not PSR-4) because WordPress file naming (`class-foo.php`) doesn't match PSR-4 conventions. The classmap is generated by `composer dump-autoload` and covers all classes in `includes/`. Helper functions in `includes/functions.php` are loaded via the `files` autoload directive.

After adding or renaming a class file, run `composer dump-autoload` to regenerate the classmap.

## Coding Standards

- Always escape global/WordPress functions with `\` when in a namespace.
- Use `use` imports for cross-namespace references — no inline `\Namespace\Class`.
- Text domain: `'atmosphere'`.
- PHP 8.1+ required.
- Follow WordPress Coding Standards (WPCS).

## Release Process

Build a release ZIP with only production dependencies:

```bash
bin/release.sh [version]
```

This runs `git archive` (respects `.gitattributes export-ignore`), installs `--no-dev` Composer dependencies with an optimised autoloader, and produces `atmosphere-{version}.zip`.

**What's included:** Plugin PHP files, templates, assets, `vendor/` with production deps only (~3 MB).

**What's excluded:** tests, bin scripts, CI config, CLAUDE.md, phpcs/phpunit config, dev dependencies (phpunit, phpcs, polyfills — ~25 MB saved).

The `.gitattributes` file controls what's excluded from `git archive` exports and GitHub release tarballs.
